Executive Summary

The extension is functionally rich and has good defensive normalization patterns in many areas, but there are critical security hygiene gaps around secret storage and persisted sensitive artifacts. A deprecated microphone subsystem is also still present as dead code, increasing attack surface and audit burden.

Highest risks: plaintext API key storage, unencrypted local sensitive data persistence, and a large unreachable mic-proxy subsystem still shipped in runtime code.

Dead Code and Structure Notes

  • Deprecated mic proxy logic still spans a large branch in background and content scripts even after runtime rejection behavior was added.
  • Approximate dead footprint: background mic branch ~369 lines; content mic-related symbol references ~160.

Remediation Plan (Multi-Job Sequence)

  1. Job 1: Remove mic proxy dead code paths and unreachable message branches end-to-end.
  2. Job 2: Replace persistent API key storage with session-only key flow (or user-secret-protected encryption).
  3. Job 3: Implement secure-at-rest mode for reports/screenshots and stronger redaction policy options.
  4. Job 4: Enforce trusted-event checks and event-rate abuse limits in capture listeners.
  5. Job 5: Minimize permissions/host scope and tighten web-accessible resource exposure.
  6. Job 6: Add strict import size caps and safe parse guards before ZIP processing.
