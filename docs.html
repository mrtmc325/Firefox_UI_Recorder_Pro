UI Recorder Pro — Documentation

Version 1.19.0. Adds Tier-2 Batches 2A + 2C on top of v1.18.0: schema-versioned storage migrations, in-memory diagnostics ring with a "Copy diagnostics" popup button, runtime-toggleable debug logs, report retention as a user setting, WebCrypto-wrapped OpenAI session key, custom redaction rules (with ReDoS-safe compile probe + runtime per-rule budget) and a live tester UI, manual redact-before-export overlay, cross-frame iframe redaction-rect coordinates (source-window verified), Ed25519-signed HTML exports with offline verifier, and an opt-in encrypted-at-rest report vault (PBKDF2-SHA-256 600k → AES-GCM-256). Batches 2B (recording presets, cross-report search, templates, undo, Markdown export, etc.) and 2D (vector annotations, perf refactors) built in worktrees during the workflow and preserved on the integration branch for follow-up.

Security & Privacy Hardening (current build)

  • Screenshot redaction policy (popup > Privacy & Stability): Omit all screenshots suppresses all screenshot pixel capture — step, lifecycle, and GIF burst frames. Steps record an explicit skip reason instead.
  • Secure-at-rest mode: reports/events are kept in browser session storage (memory-only, cleared when the browser closes) and screenshot capture is disabled. On Firefox versions without session storage, the recorder falls back to local storage with screenshots stripped. Section text/audio persistence is blocked while the mode is on. The mode is forward-only: media stored before enabling it is not purged.
  • Session-only OpenAI API key: the key lives in the report tab's session storage only — cleared when the tab closes, never written to disk, exports, or logs. Any key stored by older versions is migrated and deleted at report-page load.
  • Trusted-event capture: only real user events (isTrusted) are recorded, with per-event-type rate limits. Script-generated (synthetic) events are ignored — automation-driven flows will not produce steps.
  • Minimized permissions: host access is limited to http://*/* and https://*/* (no <all_urls>), no web-accessible resources, no about:blank frame injection.
  • Hardened raw ZIP import: strict archive-size, entry-count, and per-entry caps (sized to accept anything the extension itself exports), Store-only method, path-traversal and duplicate-name rejection, image magic-byte validation, and MIME allow-lists for restored media.
  • The deprecated microphone-capture subsystem is fully removed; residual mic requests are rejected with mic-capture-removed.

Quick Start

  1. Click the toolbar icon.
  2. Press Start.
  3. Perform your workflow.
  4. Press Stop.
  5. Open report and export as HTML or raw ZIP as needed.

Configuration How-To Templates

Template 0: Strict Active Tab Following

Use when multiple tabs/windows are open and only foreground interactions should be captured.

  • Follow active tab only (strict): On
  • Capture mode: All events
  • Auto-resume on tab focus: On
  • Limitation: when Firefox has no focused window, capture is temporarily paused until focus returns.

Template A: Dynamic Dashboard / SPA

Use for pages with live updates, modals, and in-place content refreshes.

  • Capture mode: All events
  • Page watch (dynamic UI): On
  • Page watch interval (ms): 500 to 1200
  • Diff-based screenshots: On
  • Debounce (ms): 700 to 1000

Template B: Login / Sensitive Workflow

Use for authentication and secure change-control walkthroughs.

  • Capture mode: All events
  • Redact sensitive text in report: On
  • Redact usernames on login pages: On
  • Screenshot redaction policy: Omit all screenshots (when pixels must never be captured)
  • Secure-at-rest mode: On for session-only storage; export the report before closing the browser
  • Page watch (dynamic UI): On
  • Page watch interval (ms): 600 to 1000

Template C: Long Session / Low Memory

Use for multi-hour runs or environments where memory pressure is a concern.

  • Capture mode: Clicks + submit + nav only
  • Diff-based screenshots: On
  • Debounce (ms): 900 to 1400
  • Page watch (dynamic UI): On only when needed
  • Page watch interval (ms): 1500 to 3000
  • Prune noisy input steps: On

Template D: Merge + Reorder Publishing Workflow

Use when combining multiple captures into one polished report.

  • Import raw ZIP: Merge into current report
  • Cross-tab timeline: Drag and drop to reposition timeline items
  • Step list: Use Move up / Move down for final ordering
  • Theme toggle: Choose light/dark during editing before export

Template E: Professional Client Export

Use when the final deliverable needs cleaner branding and compact structure.

  • Export theme > Palette: choose Extension Light / Cool Mist / Slate Calm
  • Export theme > Font: choose a readable family for your audience
  • Export theme > Report title size: adjust the editable title scale
  • Export theme > Report description size: tune subtitle prominence
  • Export theme > Section header theme: style exported carousel section headers and queue heading cards
  • Export theme > Section text size: tune workflow step/body text size
  • Export theme > Title/Heading themes: choose style presets for Title, Heading 1, Heading 2, and Heading 3
  • Workflow Queue: use right-side queue cards to jump sections; subtitle details reveal on hover/focus.
  • Export theme > Accent: set your team or project color

Dynamic Screenshot Rule (ui-change)

  • If page watch interval is less than 10000ms, forced screenshot interval is:
  • max(4567.38ms, pageWatchMs * 1.34562)
  • If page watch interval is 10000ms or higher, dynamic forced screenshots are disabled for ui-change.

Report Editing

  • Edit step titles and notes inline.
  • Annotate screenshots with pen/highlight/rect/outline/obfuscate/text.
  • Use Undo/Clear on annotation layers.
  • Reorder with drag/drop in timeline and Move up/Move down in the step list.
  • Use builder navigation links and panel collapse controls to reduce visual noise.
  • Use the theme icon toggle (sun/moon preview) to switch light/dark while editing.
  • Use the subsection description textarea per step to add concise context (max 200 characters, ASCII supported).
  • Use section read-aloud controls to narrate embedded text for guided walkthroughs.
  • Use Transcribe audio file in each section text panel for OpenAI speech-to-text.
  • Use Play cloud voice tour to preview OpenAI narration voices with a short fixed sample phrase (same API key and consent gate as cloud narration; no report content is sent).
  • Live microphone capture is disabled in this release.

Audio STT Flow

  • Click Transcribe audio file and pick a local audio file.
  • Grant Firefox website content consent if prompted.
  • Wait for upload/transcription to complete.
  • Review inserted text in section editor and click Save text.

Audio STT Troubleshooting

  • permission-denied: Firefox website content permission is required for OpenAI upload.
  • oversize-audio: selected file exceeds upload size limit.
  • api-failure: OpenAI returned an error; check API key and retry.
  • empty-audio: selected file has no readable audio payload.

Report Export / Import

  • Export HTML bundle: Portable carousel-style report viewer with step anchors, slide navigation, right-side Workflow Queue, and selected export theme profile.
  • Exported section header: Clean title bar with optional collapsible subsection description (no embedded URL/timestamp metadata).
  • Exported narration bar: Icon-based bottom player with timeline clock (mm:ss | mm:ss), fullscreen toggle, and gear menu for captions/fade/tempo settings.
  • Export raw ZIP: Editable payload for re-import later (bundle v4 includes frame manifest + frame files, text manifest + text payload files, and audio manifest + baked narration audio payload files).
  • Import raw ZIP: Import as new report or merge into active report.
  • Import safety: raw ZIP import enforces strict archive-size, entry-count, and per-entry caps (sized to accept anything the extension itself exports), accepts Store-only entries, and rejects unsafe or duplicate entry paths.
  • Compatibility: bundle v1 (inline screenshot ZIPs) remains supported on import.
  • Saved report rollover: Recorder keeps the 3 most recent saved reports. On each new stop/save, older reports beyond 3 are removed automatically.

Keyboard Shortcuts

  • Ctrl+Shift+Y toggles start/stop recording.
  • Hotkey stop grace: first Ctrl+Shift+Y stop press schedules stop in 2000ms; a second press during grace forces immediate stop.
  • Cmd+Opt+G (macOS target) / Ctrl+Alt+G (default) toggles high-speed GIF burst mode while recording.
  • When ON, GIF burst mode attempts screenshot capture at the selected 5/10/15 FPS target cadence until toggled OFF (suppressed while a screenshot privacy policy — omit or secure-at-rest — is active).
  • GIF burst behavior is hotkey-only; popup tuning is intentionally limited to capture FPS.
  • Capture FPS is selectable (5/10/15) in popup GIF settings; replay speed is tuned in the report builder and carried into export.
  • Replay timing uses measured source FPS per burst as the 1.0x baseline, so playback pace matches recorded cadence.
  • Popup/API stop actions remain immediate and do not use the hotkey stop grace delay.
  • If this shortcut conflicts with a browser shortcut, rebind the extension command from Firefox extension shortcuts.

Privacy

All core capture/report data remains in local browser storage; with secure-at-rest mode on, recorder events/reports are kept in memory-only browser session storage instead. Only trusted (real user) events are recorded — script-generated events are ignored. Optional OpenAI cloud narration sends section text only when explicitly selected in the builder with a provided API key and granted Firefox website content permission; the key lives only in the report tab's session storage and is never persisted to disk. Optional OpenAI speech-to-text uploads user-selected audio files only after the same explicit key/permission flow. Text redaction applies to report text fields; captured screenshot pixels are not edited — use the screenshot redaction policy (omit) or secure-at-rest mode to suppress screenshot capture entirely for sensitive workflows.
